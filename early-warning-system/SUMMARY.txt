STEP 2 — ML MODEL TRAINING (train_model.py)
--------------------------------------------
File created: backend/ml/train_model.py

What it does:
  1. Generates 500 synthetic student records with 5 features:
       - attendance       (range: 50–100)
       - internal_marks   (range: 0–100)
       - assignment_score (range: 0–100)
       - lms_activity     (range: 0–100)
       - stress_score     (range: 0–100)

  2. Labels each student as at_risk = 1 if ANY of:
       attendance < 65  OR  internal_marks < 50  OR
       assignment_score < 50  OR  lms_activity < 40  OR
       stress_score > 70
     otherwise at_risk = 0

  3. Trains a Logistic Regression model (scikit-learn):
       - 80/20 train-test split (stratified)
       - StandardScaler for feature normalization
       - LogisticRegression (max_iter=1000)

  4. Result: Model Accuracy = 97.00%

  5. Saves model + scaler + feature list to:
       backend/ml/risk_model.pkl

  To retrain anytime:
    python ml/train_model.py


STEP 3 — PREDICTION ENDPOINT (predictions.py)
----------------------------------------------
File updated: backend/routes/predictions.py

What it does:
  1. Loads risk_model.pkl at startup (model + scaler + features)
  2. Accepts POST /predictions/predict with JSON body:
       { attendance, internal_marks, assignment_score,
         lms_activity, stress_score }
  3. Scales input → runs predict_proba() → gets class-1 probability
  4. Maps probability to risk level:
       < 0.4  → Green
       0.4–0.7 → Yellow
       > 0.7  → Red
  5. Returns: { "risk_score": 0.xxxx, "risk_level": "Green/Yellow/Red" }

Also updated:
  - main.py → registered students + predictions routers
  - CORS → allow both localhost:5173 and localhost:5174
